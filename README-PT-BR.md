## Big O Notation — Análise de Complexidade

A **Big O Notation** mede o **crescimento do tempo ou espaço** que um algoritmo consome conforme o tamanho da entrada (`n`) aumenta.  
Ela é essencial para avaliar a **eficiência** das soluções do LeetCode.

### Complexidade de Tempo

| Notação | Tipo | Exemplo de Algoritmo |
|----------|------|----------------------|
| O(1) | Constante | Acesso direto em array |
| O(log n) | Logarítmica | Busca binária |
| O(n) | Linear | Percorrer um vetor |
| O(n log n) | Linearítmica | Merge Sort, Quick Sort |
| O(n²) | Quadrática | Bubble Sort |
| O(2ⁿ) | Exponencial | Fibonacci recursivo |
| O(n!) | Fatorial | Geração de permutações |

### Complexidade de Espaço

| Notação | Tipo | Exemplo |
|----------|------|---------|
| O(1) | Constante | Variáveis fixas |
| O(n) | Linear | Estruturas proporcionais ao tamanho da entrada |
| O(n²) | Quadrática | Matrizes bidimensionais |
